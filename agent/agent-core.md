# Agent Core（统一规则源）

## 目标
- 保持最小改动
- 先读后改
- 先测后交付
- 明确影响范围与回滚路径

## 工作流
1. 读取上下文与约束
2. 判断任务域并按需读取 `.skills/` 下相关 Skill
3. 形成最小变更方案
4. 先改测试再改实现（可行时）
5. 优先在 Dev Container 中执行最小验证
6. 涉及结构调整时，同时更新 BUILD、模块路径、CI/部署与文档
7. 输出：变更、验证、风险、后续建议

## Skill 路由
- `development-environment`：开发环境一致性、性能与安全
- `dependency-build`：依赖与构建图谱、目标粒度、验证范围
- `architecture-planning`：目录拓扑、边界治理、跨模块依赖
- 多域任务先确定主域，再按需补充其他 Skill

## 环境约束
- 默认不要求本机安装依赖
- 验证命令优先通过容器执行
- 优先使用可复用缓存，减少重复下载与重复构建
- 环境改动同时评估一致性、速度与安全

## Monorepo 约束
- services: 应用与服务
- packages: 复用库
- tools: 工具与脚本
- 仓库默认只提交源码与资源，构建产物、IDE 本地文件、临时方案文件应缩小作用域并加入忽略规则
- 跨目录依赖需说明原因
- 新增依赖优先通过仓库统一构建系统管理
- 结构规划优先保证清晰边界，避免短期便利固化为长期耦合

## 输出格式
- 先结论，后细节
- 包含：修改文件列表、命令、结果、风险
- 如果使用了 Skill，说明使用了哪个 Skill，以及它影响了哪些决策
- 未验证项必须明确标记
