# 开发环境

## 何时使用
- 需要搭建或调整本地/容器开发环境时。
- 需要提升可复现性、启动/构建速度或缓存复用时。
- 需要配置代理、镜像源、证书或网络相关构建行为时。
- 需要加强容器安全、权限控制或供应链暴露面治理时。

## 核心流程
1. 修改前先阅读现有环境文档和配置。
2. 优先选择最小改动以提升团队环境一致性。
3. 明确并稳定缓存策略（镜像层缓存与运行时缓存分开考虑）。
4. 采用安全优先默认：最小权限、禁止硬编码密钥、最小化网络暴露面。
5. 在目标环境执行最小验证，并标注未验证范围。

## 检查清单
- 可复现性：同一命令在全新环境中应得到一致结果。
- 性能：缓存路径/挂载应稳定且可复用。
- 安全：凭据外置，特权操作最小化。
- 作用域控制：不把构建产物或机器本地临时文件提交到版本库。
- 文档一致性：使用方式变更需同步到项目文档。

## 验证示例
- 环境与工具链版本检查。
- 依赖图或锁文件解析检查。
- 运行项目 CI 路径中的构建与测试命令。

## 输出要求
- 列出修改文件及修改原因。
- 说明对一致性、速度和安全的影响。
- 提供已执行验证命令与明确未验证项。

## 补充说明
- 这个技能用于处理开发环境一致性、容器化工作流、缓存策略、构建速度和容器安全。
- 修改环境时，优先保持缓存路径、命名卷和构建入口一致，避免不同开发者得到不同结果。
- 中文说明放在正文里，便于团队阅读；`name` 和 `description` 继续保留英文，方便各类 Code Agent 发现。
